# 手把手部署运行教程（开发板实测）

从「PC 上一份刚生成好代码的工程」开始，**一步步把 C 代码传到开发板（树莓派等任意 Linux 板）、
编译、声学实测、取回结果、在 MATLAB 里解码**，跑出实验一的 1kHz 频谱峰值和实验二的 BER=0。

> 本教程以**树莓派**为例，但对任意装了 `libasound`(ALSA) 的 Linux 板（Jetson / 香橙派 / …）
> 完全一样，差别只有一个 `-d <声卡设备名>`。踩坑速查见 [Q&A](Q&A.md)。

---

## 0. 角色与约定

| 角色 | 说明 |
|---|---|
| **[PC]** | 你的宿主机。装 **MATLAB**（负责放音 `bok_emit`/`single_fre_emit` 与解码 `bok_rev`/`spectrum`）。Windows 上再装 **Git Bash** 以便用 `ssh`/`scp`/`tar` 等 Linux 命令。 |
| **[板]** | 开发板。`ssh` 登进去操作。需 Linux + 麦克风 + `gcc`。 |

命令前缀标明在哪敲。占位符按你的实际情况替换：

- `<user>@<board-ip>` —— 板子登录账号与 IP（示例：`padhia@192.168.3.82`）
- `plughw:X,0` —— 麦克风声卡号，**用 `arecord -l` 现查**（示例：树莓派上 USB 麦克风常是 `plughw:2,0`）

> **重要原则：放音用 PC、录音用板子，不要自放自录**；而且**先在板上启动录音、再到 PC 放音**
> （`-t` 是固定录音窗口，放早了帧头会丢）。

---

## 1. 一次性准备

### 1.1 [板] 安装 ALSA 开发库（编译需要 `-lasound`，只需一次）
```bash
ssh <user>@<board-ip> 'sudo apt update && sudo apt install -y libasound2-dev'
# Fedora 系: sudo dnf install alsa-lib-devel
```

### 1.2 [PC]（可选但推荐）配 ssh 免密，省得每步输密码
```bash
ssh-keygen -t ed25519        # 一路回车（已有可跳过）
ssh-copy-id <user>@<board-ip>
```
（`ssh-keygen`/`ssh-copy-id` 在 Linux/macOS 终端与 Windows Git Bash 里都有。）

---

## 2. 把 C 代码传到开发板（tar over ssh，只传源码、保留目录层级）

工程里 MATLAB 模型(`.slx`)、生成缓存(`slprj`)、发射/解码脚本(`host/`)、图片
都**不需要**上板。下面这条命令**只挑 `Makefile` + `.c` + `.h`**，打包经 ssh 解到板上，
并**保持 `common/ exp1/ exp2/` 同级层级**（Makefile 用 `../common` 引用共享底层）：

**[PC]** 在工程根目录执行：
```bash
find common exp1_single_freq exp2_chirp \
     \( -name '*.c' -o -name '*.h' -o -name 'Makefile' \) \
     -not -path '*/slprj/*' \
  | tar czf - -T - \
  | ssh <user>@<board-ip> 'mkdir -p ~/portable-acoustic-sdr && tar xzf - -C ~/portable-acoustic-sdr'
```

> **为什么用 tar over ssh 而不是 rsync**：`rsync` 默认**不在 Windows 的 Git Bash 里**
> （需手动加装），而 `find`/`tar`/`ssh` 都是 Git Bash 自带；这条命令在 Linux/macOS/Windows
> 三种宿主上**完全一致**，所以教学统一用它。

传完后板上应是（只有源码，干净）：
```
~/portable-acoustic-sdr/
├── common/{include/*.h, src/*.c}
├── exp1_single_freq/{Makefile, include/*.h, src/*.c, simulink_model/single_fre_rev_ert_rtw/*.c,*.h}
└── exp2_chirp/{Makefile, include/*.h, src/*.c, simulink_model/chirp_rev_detect_ert_rtw/*.c,*.h}
```

> 以后改了算法、在 PC 上重新生成模型，**再跑一次这条命令**即可（tar 会覆盖旧文件）。

## 3. 实验一 · 单频信号

### 3.1 [板] 编译 + 确认麦克风声卡号
```bash
ssh <user>@<board-ip>
cd ~/portable-acoustic-sdr/exp1_single_freq
make clean && make            # 换平台必须先 clean，否则旧 .o 时间戳会误判已最新
arecord -l                    # 看麦克风是 card 几，记下 X
```

### 3.2 声学实测（先录后放）
**[板]** 先启动采集（阻塞约 8 秒，给你切到 MATLAB 的时间）：
```bash
./build/sdr_rx -d plughw:X,0 -t 8       # X 按 arecord -l 实际改
```
**[PC · MATLAB]** 看到板子开始录后，**立刻**运行发射脚本播 1kHz 单音：
```matlab
cd exp1_single_freq/host
single_fre_emit
```

### 3.3 [PC] 取回 .mat + 看频谱
```bash
scp <user>@<board-ip>:~/portable-acoustic-sdr/exp1_single_freq/single_f.mat exp1_single_freq/
```
**[PC · MATLAB]**：
```matlab
cd exp1_single_freq
spectrum('single_f.mat','toFileData')
```
✅ **频谱峰值落在约 1000 Hz** → 实验一通过。

## 4. 实验二 · chirp 扩频

### 4.1 [板] 编译
```bash
ssh <user>@<board-ip>
cd ~/portable-acoustic-sdr/exp2_chirp
make clean && make
arecord -l                    # 确认声卡号 X
```

### 4.2 [PC · MATLAB] 选定图片（发射端与解码端必须一致）
分别打开 `exp2_chirp/host/bok_emit.m` 和 `bok_rev.m`，把顶部
`img_name` 改成同一张（默认 `ren128b.bmp`，最短约 18s，先用它跑通）。

### 4.3 声学实测（先录后放）
**[板]** 先启动采集（`ren128b` 约 18s，给到 `-t 24` 留余量；大图按 `bok_emit` 打印的“时长”加几秒）：
```bash
./build/chirp_rx -d plughw:X,0 -t 24
```
**[PC · MATLAB]** 立刻运行发射脚本（它会显示发送图、用 `sound()` 播放，并把 `info_all.mat` 写到 `host/`）：
```matlab
cd exp2_chirp/host
bok_emit
```

### 4.4 [PC] 取回判决数据 + 解码算 BER
```bash
scp <user>@<board-ip>:~/portable-acoustic-sdr/exp2_chirp/chirp5.mat exp2_chirp/host/
```
**[PC · MATLAB]**（`chirp5.mat` 与 `info_all.mat` 都在 `host/`，`bok_rev` 自动读）：
```matlab
cd exp2_chirp/host
bok_rev
```
✅ **命令窗口 `BER = 0`、`imshow` 还原出正确点阵** → 实验二通过。

## 5. 重新生成模型后如何重新部署

改了 Simulink 模型、在 PC 上重新生成了 C 代码（步骤见
[实验一](实验一_单频信号.md)/[实验二](实验二_chirp扩频.md)的"重新生成模型"），
按下面重新部署即可。

### 5.1 [PC] 重传代码（同一条 tar over ssh，自动覆盖）
和[第 2 节](#2-把-c-代码传到开发板tar-over-ssh只传源码保留目录层级)**完全相同**的命令再跑一次。
`tar` 解包**默认覆盖**板上同名文件，所以改过的 `.c/.h` 会被盖成最新：
```bash
find common exp1_single_freq exp2_chirp \
     \( -name '*.c' -o -name '*.h' -o -name 'Makefile' \) \
     -not -path '*/slprj/*' \
  | tar czf - -T - \
  | ssh <user>@<board-ip> 'mkdir -p ~/portable-acoustic-sdr && tar xzf - -C ~/portable-acoustic-sdr'
```

> ⚠️ **若这次重新生成让生成的 `.c` 文件改了名或减少了**（不只是内容变化），
> tar 只会覆盖/新增、**不会删除**板上旧文件，残留的"孤儿 `.c`"会被 Makefile 的
> `*.c` 通配编进去导致报错。此时**先删掉板上对应的生成目录再重传**：
> ```bash
> # 实验二为例；实验一对应 single_fre_rev_ert_rtw
> ssh <user>@<board-ip> 'rm -rf ~/portable-acoustic-sdr/exp2_chirp/simulink_model/chirp_rev_detect_ert_rtw'
> ```

### 5.2 [板] 清干净重新编译（**务必 `make clean`**）
```bash
ssh <user>@<board-ip>
cd ~/portable-acoustic-sdr/exp2_chirp     # 或 exp1_single_freq
make clean && make
```
**`make clean` 不能省**：PC 与板子时钟可能有偏差，若新 `.c` 的时间戳不比旧 `.o` 新，
`make` 会误判"已是最新"而不重编，跑的还是旧逻辑。clean 一下最稳。

### 5.3 接口改了名时同步胶水层
如果重新生成时改了 **Inport/Outport 的名字**，只需改 `src/model_glue.c` 这**一处**
对应的字段名——它也在上面 tar 传输的 `.c` 里，**在 PC 上改好再重传**即可。

### 5.4 照常运行
回到[第 3 节](#3-实验一--单频信号)/[第 4 节](#4-实验二--chirp-扩频)的"声学实测"重新跑一遍验证。

## 6. 跑不通时

按「现象 → 原因 → 解决」整理在 [Q&A](Q&A.md)，最常见的几条：

- `capture slave is not defined` / 打不开 `default` → 没带 `-d`，用 `-d plughw:X,0`（[Q1/Q2](Q&A.md)）。
- 判决恒 −1 / 帧同步失败 → 多为声道问题，接收端已**默认单声道 `-c 1`**，一般无需改（[Q3](Q&A.md)）。
- 录到静音（RMS 个位数）→ 麦克风没接好，非软件问题（[Q4](Q&A.md)）。
- 大图解不出 → `-t` 不够，按 `bok_emit` 打印的时长加足（[Q6](Q&A.md)）。
- `make` 报找不到 `asoundlib.h`/`-lasound` → 没装 `libasound2-dev`（[Q9](Q&A.md)）。
- `git push` / ssh 认证问题、card 号变动等 → 见 [Q&A](Q&A.md)。

## 附：宿主机差异速记

| 环节 | Linux / macOS 宿主 | Windows 宿主 |
|---|---|---|
| 跑 `find`/`tar`/`ssh`/`scp` | 自带终端 | **Git Bash**（自带这些命令） |
| 传代码到板子 | 本教程 tar over ssh | **同左**（rsync 没有，统一用 tar） |
| 放音 / 解码 | MATLAB `bok_emit`/`bok_rev`/`single_fre_emit`/`spectrum` | **同左**（都在 MATLAB 里） |
| 路径写法 | `/home/you/...` | Git Bash 里盘符为 `/c/Users/...` |
