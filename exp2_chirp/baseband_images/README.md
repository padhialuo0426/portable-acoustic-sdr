# 基带图像（chirp 扩频通信的待传信息）

实验二的系统模型：

```
图像(基带信号) → 比特流 → BOK chirp 调制(载波 1kHz 声音) → 扬声器(发射)
                                                              ↓ 声波信道
        还原图像 ← 解码/判决 ← chirp 接收检测(本工程) ← 麦克风(接收天线)
```

- **基带信号**：下面这些 1-bit 位图，逐像素展开成比特流。
- **载波**：1000 Hz 音频。比特经 BOK（二进制正交键控）映射为上扫/下扫 chirp：
  比特 0 → 上扫(μ=+1)，比特 1 → 下扫(μ=−1)；T=0.1s/符号，B=200Hz，fs=8000。
- **发射**：MATLAB `Bok_emit.m` 用 `sound()` 经扬声器播放调制波形。
- **接收**：麦克风采集 → 本工程 `chirp_rx` 做匹配滤波/相关检测 → 输出每帧判决值
  (`chirp5.mat/toFileData5`) → `Bok_rev.m` 帧同步、硬判决、算 BER、`imshow` 还原图像。

## 图像清单

| 文件 | 尺寸(宽×高) | 位数 | 说明 |
|---|---|---|---|
| `ren128b.bmp` | 16×8 | **128** | ★ 当前 `Bok_emit.m` 实际发送的图（帧载荷正好 128 位） |
| `ren512b.bmp` | 32×16 | 512 | 更大版本，当前帧装不下，未发送 |
| `lan336.bmp` | 21×16 | 336 | 备选图，未发送 |
| `lzu2048b.bmp` | 64×32 | 2048 | **「兰州大学」字样**；2048 位，远超当前 128 位帧，未发送 |

## 帧结构（180 符号，对应 `Bok_emit.m`）

```
[1:10]   全 0           静默间隔
[11:20]  1010101010     交替同步前导
[21:35]  m序列(15位)     帧头同步  [1 0 0 1 1 0 1 0 1 1 1 1 0 0 0]
[36:163] 图像 128 位     ★ 基带载荷 (ren128b.bmp)
[164:178] m序列(15位)    帧尾同步
```

> 注：当前帧只承载 128 位，故只有 `ren128b.bmp` 能直接发送。若要传「兰州大学」
> (`lzu2048b.bmp`, 2048 位)，需改大 `Bok_emit.m` 的符号数 `code` 与帧结构，
> 接收端无需改动（`chirp_rx` 只管输出判决流，解码在 PC 端 `Bok_rev.m`）。
