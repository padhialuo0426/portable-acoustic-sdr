# 单频接收模型（适配当前 MATLAB 版本）

`single_fre_rev.slx` 是从原树莓派工程改造而来、用于通用 Linux 的模型。
生成的纯算法 C 代码就在同目录的 `single_fre_rev_ert_rtw/` 下，Makefile 直接读取，
**无需手动拷贝**。

## 已做的改造（相对原树莓派模型）

1. `ALSA Audio Capture` 块 → **Inport `AudioIn`**（`int16`, 维度 `[80 2]`，
   去交织布局 `[L0..79, R0..R79]`）。音频采集移到模型外，由板级 POSIX 代码喂入。
2. 两个 `To File` 块 → **Outport `out_f1` / `out_f2`**（`double[80]`）。
   输出移到模型外，由板级 `mat_sink.c` 写成 `.mat`。
3. 配置：`System target file = ert.tlc`，`Hardware board = None`，`Language = C`，
   `GenCodeOnly = on`，`MatFileLogging = off`（避免 To File 拖入 rt_logging）。

## 如何重新生成（修改算法后）

在 MATLAB 里：

```matlab
cd <本目录>            % exp1_single_freq/matlab
load_system('single_fre_rev')
% ……（如需改算法在此修改）……
set_param('single_fre_rev','SystemTargetFile','ert.tlc');
set_param('single_fre_rev','HardwareBoard','None');
set_param('single_fre_rev','GenCodeOnly','on');
set_param('single_fre_rev','MatFileLogging','off');
slbuild('single_fre_rev');     % 代码直接生成到 single_fre_rev_ert_rtw/
```

生成完成后回到 `exp1_single_freq/`，直接 `make` 即可——Makefile 自动读取
`single_fre_rev_ert_rtw/*.c`。生成目录里的 `.mk / .mat / html / tmwinternal`
等是缓存/工程文件，已被 `.gitignore` 忽略，不影响编译。

## 若改动了 Inport/Outport 名字

`../src/model_glue.c` 里引用了外部输入输出的字段名
（`single_fre_rev_U.AudioIn`、`single_fre_rev_Y.out_f1/out_f2`）。
若重新生成后这些名字变了，只需同步改 `model_glue.c` 这一处。
